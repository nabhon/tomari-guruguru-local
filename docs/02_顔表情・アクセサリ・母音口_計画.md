# Plan: Accessories, Emotion Symbols, and Vowel Mouths

## Context

The user wants the avatar (トマリトーク) to do three new things:
1. **Toggleable PNG accessories** (glasses, hats, …) layered on the character.
2. **Emotion symbols** — auto-detected facial emotion shown as anime-style emote overlays (no face-art change).
3. **Vowel mouths (a/i/u/e/o)** — speech-shaped mouth driven by the face camera.

Key constraint discovered during exploration: the character is a **fixed set of pre-rendered sprites** — 6 sheets `A–F` = eyes(open/closed) × mouth(closed/half/open), each a 5×5 grid of 25 head directions (150 frames). MediaPipe FaceLandmarker already produces 52 blendshapes but only `jawOpen` + `eyeBlink*` are used today, and the audio path is amplitude-only (no formants).

So **detection** is cheap for all three features, but **display** differs:
- Accessories and emotion symbols are pure overlay layers → no art regeneration.
- True vowel mouths need 5 mouth shapes the built-in art doesn't have. We therefore build the vowel **detection + display pipeline now, asset-ready**, with a **fallback** that maps a/i/u/e/o onto the existing 3 mouth states so it works immediately with built-in art and "lights up" when a character ships vowel sheets.

User decisions: accessories use a **single PNG + per-direction offset table**; emotions show as **anime emote symbols** (not face art); vowels are detected from **face blendshapes only**.

## New files

### `src/overlay-config.js` (pure data + helpers, no React)
- `ACCESSORIES`: `[{ id, name, file, anchor }]`, `file` public-relative (e.g. `'accessories/glasses.png'`), `anchor: 'eyes'|'head'`.
- `EMOTES`: `[{ id, emotion, file, corner }]` mapping emotion enum → emote PNG (e.g. `'emotes/blush.png'`) + fixed position (`corner:'tr'`, etc.).
- `buildOffsetTable(anchor)` → 25-entry array (index `r*5+c`) of `{dx,dy,scale,rot}`, derived linearly: center `(2,2)`=`{0,0,1,0}`, `dx=(c-2)*kx`, `dy=(r-2)*ky` as **fractions of the character box** (`translate(%)`), small scale/rot easing toward edges.
- `OFFSET_OVERRIDES`: optional per-accessory partial tables merged over the default for hand-tuning.
- `offsetFor(accId, r, c)` → merged `{dx,dy,scale,rot}`.

### `src/components/OverlayLayer.jsx` (one component, two item types)
- ESM import in `talk-app.jsx` (NOT a `window.*` global — that list is only for the `Tweak*` family).
- A single `position:absolute; inset:0` container rendered **after the `allFrames` img stack but inside `charRef`** (talk-app.jsx:264–277) so it shares the coordinate box, survives F9 hide-UI, and is captured by greenscreen. `zIndex:2` (img stack is default 0).
- **Accessories** (offset-tracked): for enabled accessories, look up `offsetFor(id, cell.r, cell.c)`, apply `transform: translate(dx%,dy%) scale(scale*accScale) rotate(rot)`; `dx/dy` multiplied by `accOffsetGain`. Only enabled accessories mount.
- **Emote symbols** (corner-anchored): mount only when `showEmotes && emotion === item.emotion`; opacity/scale from `emotionIntensity`; brief pop via a locally injected `<style>` keyframe (same technique as tweaks-panel.jsx).
- Asset URLs: `` `${import.meta.env.BASE_URL}${item.file}` `` (resolves dev `/`, Pages `/tomari-guruguru/`, electron `./`).

### Public assets
- `public/accessories/*.png`, `public/emotes/*.png`. Not asserted by `verify:pages` (it only checks `slices2/A–F` + mediapipe), so no script change.

## Existing files to modify

### `src/drivers/faceTracking.js`
All continuous reads go in `handleResult` (~93–99) reusing `scoreOf(categories,name)`:
- Capture into refs: `mouthSmileL/R`, `mouthFrownL/R`, `eyeWideL/R`, `browOuterUpL/R`, `browInnerUp`, `browDownL/R`, `noseSneerL/R`, `mouthFunnel`, `mouthPucker`, `mouthClose` (jawOpen already in `jawRef`).
- **Emotion detection** → `emotionRef` + debounced `setEmotion`/`setEmotionIntensity` (enum `neutral|happy|surprised|angry|sad`). Scores: happy=`smile`; surprised=`min(eyeWide,jaw)+0.5*browUp`; angry=`min(browDn,sneer)`; sad=`min(frown,browInUp)`. Argmax above `tw.emotionThreshold`; **~250ms** debounce (calmer than mouth); only `setState` on change and quantize intensity (round 0.05) to avoid per-frame re-renders.
- **`frameVowel(now, tw)`** (pull-based, mirrors `frameMouth` at 103–112, own `lastVowel`/`lastVowelSwitch` refs, 70ms debounce). Enum **0=closed,1=a,2=i,3=u,4=e,5=o**:
  - `jaw<jawHalf && funnel<vowelFunnel` → 0
  - `funnel>=vowelFunnel || pucker>=vowelFunnel` → `jaw>=jawFull ? 5(o) : 3(u)`
  - `smile>=vowelSmile` → `jaw>=jawHalf ? 4(e) : 2(i)`
  - else → `jaw>=jawHalf ? 1(a) : 0`
- Add `frameVowel, emotion, emotionIntensity` to the hook's return object.

### `src/character-config.js`
- Add `vowelSheets: { eyesOpen: null, eyesClosed: null }` (built-in = no vowel art). A vowel-capable custom character would set e.g. `eyesOpen:{a:'G',i:'H',u:'I',e:'J',o:'K'}`, `eyesClosed:{a:'L',…,o:'P'}`.
- Add `hasVowels()` → `!!(vowelSheets.eyesOpen && vowelSheets.eyesClosed)`.
- `srcFrom`/`src`/`basePath`/`rows`/`cols`/`ext` unchanged — `srcFrom` already builds paths for any sheet letter.
- **Built-in stays A–F only**; vowels are opt-in per custom character → `verify:pages` unchanged.

### `src/talk-app.jsx`
1. **EDITMODE defaults** (14–43, keep valid JSON literal) — add keys in section below.
2. **`sheetFor`** (131): when vowel mode active AND `charConfig.hasVowels()` → pick from a `VOWEL_SHEETS[eyesClosed?'eyesClosed':'eyesOpen'][vowelLetter]`; else `SHEETS[(eyesClosed?3:0)+m]` where `m` is a 0/1/2 sheet-mouth.
3. **Vowel→sheet mapping**: `const VOWEL_TO_MOUTH = [0,2,1,1,2,1]` (closed→0, a→open, i→half, u→half, e→open, o→half). Helper `vowelToSheetMouth(m,tw)`: pass vowel enum through when `tw.vowelMouth && charConfig.hasVowels()`; else collapse via `VOWEL_TO_MOUTH`. Audio/jaw results are already 0/1/2 and pass through untouched.
4. **`onFrame`** (219–222) — add vowel branch with precedence:
   ```
   let m;
   if (tw.vowelMouth && face.cameraOn)        m = face.frameVowel(now, tw);   // 0..5
   else if (tw.mouthFromFace && face.cameraOn) m = face.frameMouth(now, tw);  // 0..2
   else                                        m = audio.frame(now, tw);      // 0..2
   if (m != null) setMouth(vowelToSheetMouth(m, tw));
   ```
   Keep `mouth` state in the **sheet-mouth domain** so `activeSheet`/`allFrames` logic stays localized.
5. **`allFrames`** (243–247) — iterate `ALL_SHEETS = charConfig.hasVowels() ? [...SHEETS, ...vowelLetters] : SHEETS`. **Only include vowel letters when present** (avoid 404s on built-in). Add `charBase` (or a `hasVowels` boolean) to the `useMemo` deps (currently `[]`) so character switching rebuilds the frame set.
6. **Render overlay** after the img stack, inside `charRef` (~276):
   ```
   <OverlayLayer charBase={charBase}
     accessories={ACCESSORIES.filter(a => t[`acc_${a.id}`])}
     cell={cell} mouth={mouth} blink={blink}
     emotion={face.emotion} emotionIntensity={face.emotionIntensity}
     showEmotes={t.emotionSymbols && face.cameraOn} tweaks={t} />
   ```
7. **TweaksPanel JSX** — add controls under new `TweakSection`s ("Accessories", "Emotion symbols") and a vowel toggle near the existing mouth-from-face controls.

## Risks / edge cases
- **404 preloading (critical):** gate vowel letters behind `hasVowels()` in `ALL_SHEETS`; add character signal to the `allFrames` `useMemo` deps (today `[]`).
- **Capture/greenscreen:** overlay must live inside `charRef` (outside the `!hideUI` block) so accessories survive F9 + chroma. Emotes also inside `charRef`, gated on `face.cameraOn`.
- **Re-render churn:** only `setEmotion`/`setEmotionIntensity` on change; quantize intensity.
- **Debounce isolation:** `frameVowel` uses its own last-state refs so mode switches don't carry stale `frameMouth` state.
- **deviceId persistence:** unchanged — `cameraId` intentionally not persisted; add nothing that persists it.
- **EDITMODE JSON:** new keys must stay plain JSON (flat booleans/numbers), markers intact.
- **Offset units:** percentages of the character box (not px) so accessories track across `charSize`/`vmin` changes.

## Out of scope (called out for the user)
- **Vowel-accurate art:** to make a/i/u/e/o look truly distinct you must generate vowel mouth sheets (G–P) per character via the existing sheet pipeline; until then the fallback (open/half/closed) is what shows.
- **Emotion face art** and **audio-formant vowel detection** were explicitly excluded.
